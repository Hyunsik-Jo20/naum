# 동네 기상(기상청 동네예보) 연동 설정

상단 날씨 바를 **기상청 동네예보(초단기실황)** + **에어코리아(미세먼지)** 실시간으로 연결하는 방법.
키가 없으면 자동으로 Open-Meteo로 동작하므로, 키를 받은 뒤 아래만 설정하면 됩니다.

## 1. 인증키 발급 (공공데이터포털)

1. https://www.data.go.kr 회원가입/로그인
2. **기상청_단기예보 ((구)_동네예보) 조회서비스** 검색 → **활용신청**
   - 제공 함수 중 `getUltraSrtNcst`(초단기실황) 사용
3. **한국환경공단_에어코리아_대기오염정보** 검색 → **활용신청**
   - `getCtprvnRltmMesureDnsty`(시도별 실시간 측정정보) 사용
4. 신청 승인 후 **마이페이지 → 인증키**에서 **Encoding 인증키** 복사
   - (보통 신청 직후~1시간 내 사용 가능)

## 2. 프로젝트 설정

1. `.env.example` 를 복사해 **`.env.local`** 생성
2. 값 입력
   ```
   VITE_WEATHER_SOURCE=kma
   DATAGOKR_KEY=<복사한 Encoding 인증키>
   ```
3. **개발 서버 재시작** (`vite.config.ts`/env 변경은 재시작 필요)
   ```
   npm run dev
   ```

## 3. 동작 방식

- 브라우저 → `/api/kma`, `/api/air` (개발 프록시) → data.go.kr
  - 프록시가 `serviceKey`를 **서버측에서 주입** → 키가 브라우저 번들에 노출되지 않음
  - data.go.kr의 CORS 미허용 문제도 프록시로 해결
- 학교 좌표(`src/data/location.ts`의 `SCHOOL`)를 기상청 격자(nx, ny)로 변환해 **동네 단위** 실황 조회
- 미세먼지는 에어코리아 시도(부산) 측정소 평균
- 호출 실패(키 미설정/지연 등) 시 **Open-Meteo로 자동 폴백** → 화면은 항상 동작

## 4. 운영(배포) 주의

- 위 프록시는 **개발 서버 전용**입니다. 배포 시에는 동일 역할의 **서버(API 프록시)**가 필요합니다.
  - 설계의 "서버 계층"에서 기상청/에어코리아를 호출하고 키를 보관하도록 구현하면 됩니다.
- 과거 14일(교육청 날씨 연계 분석)은 Open-Meteo 히스토리를 사용합니다(기상청 단기예보는 과거 일별 제공이 제한적).

## 5. 실제 학교로 바꾸기

`src/data/location.ts` 의 `SCHOOL`(이름·시도·위경도)을 실제 학교 값으로 수정.
