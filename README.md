# 나음(NaUM) — 보건실 디지털 전환 플랫폼

> 학생이 보건실에 오면 **본인이 직접 증상을 접수**하고, 보건교사는 **처치에만 집중**하며,
> 담임·학부모는 **접수·종료 상태를 실시간으로** 확인하는, **개인정보가 서버에 남지 않는** 보건실 운영 플랫폼.
>
> 코드명 NaUM은 가칭이며 명칭은 검토 예정.

## 핵심 원칙
- **개인정보 로컬 격리:** 이름·반·번호·연락처는 보건실 로컬에만, 서버는 비식별 데이터(난수 토큰·학년·성별·병명계통·시각)만.
- **무문자 단순성:** QR 찍기 → 그림 누르기 → 완료. 글 못 읽는 저학년도 가능.
- **기록 시작점 이동:** 사후 기억 입력 → 실시간 셀프 접수 + 자동 시각 캡처.
- **자동화:** 월간 교장 보고서·통계를 비식별 데이터로 자동 생성.

## 단계
1. **MVP(단일 학교):** 키오스크 접수 / 보건교사 처치 / 담임·학부모 알림(접수·종료) / 월간 보고서 자동
2. 단일교 통계 고도화·반복방문 패턴
3. 교육청 다중학교 집계 + 카카오맵 + 감염병 신호 + 양방향 공지
4. 날씨·지형 외부데이터 연계 예방·예측

## 상태
설계 확정(2026-06-13). **프로토타입 1차 완료 — 5개 핵심 화면 동작.**

### 프로토타입 실행
```
npm install
npm run dev   # http://localhost:5173
```

### 구현된 화면
- 학생 키오스크 접수 (QR 시뮬/이름 선택 → 증상 다중선택 → 대기번호)
- 보건교사 콘솔 (좌 대기 · 가운데 항상 열린 처치화면 · 우 종료, 완료 시 다음 대기자 자동 오픈, 종료자 사후 보완)
- 담임·학부모 알림 (접수·종료 2회, 담임/학부모 보기)
- 교육청 대시보드 (필터·KPI·계통별·이상신호·공지, 지도는 카카오맵 자리 모형)

스택: React + Vite + TypeScript + react-router.

### 로컬/서버 분리 (모형)
- **로컬(PII):** 학생 명부 `src/data/roster.ts` + `visit_id ↔ student_id` 매핑(store 내부). 화면은 `studentOf()`로 이름 복원.
- **서버(비식별):** 방문 `Visit`에는 학년·성별·증상·병명계통·처치·상태·시각만 — 이름·반·번호 없음.

### 더미 학생 명부
- `local-data/students.xlsx` — 학교 로컬 운영용 엑셀 (초등 6학년, 학년별 2~3반, 반당 20~25명, 남 1~19번·여 30~50번)
- 재생성: `python scripts/gen_roster.py` (→ xlsx + roster.ts 동시 생성)
